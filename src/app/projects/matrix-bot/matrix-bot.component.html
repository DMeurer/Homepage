Matrix Bot

This is a simple chatbot for the Matrix protocol. My previous chatbot was running on telegram, simply because whatsapp didnt allow it. But when I started to host my own Matrix instance, I had created a new "main" chat app for me. And more importantly, one where I can do whatever I want.

The project uses matrix-sdk for chat, reqwest and scraper for HTTP scraping, and SQLite for persistence.

What it does

The functionality is a bit different from what I previously had on the telegram bot. Simply because the canteen menu was the only thing I really used. So only the canteen menu is transferred to the new codebase.

Something new is the alert system. This allows to set alerts on websites. The bot will then use polling to check the page, and send a message if anything changes. This system was designed for me and my friends on the matrix instance, so creating alerts will require extra (bot admin) permissions. Additionally, all (permitted) users can see and change all alerts.

Example alert flow

Configuration

The bot is configured through environment variables. The important part is the Matrix login and the optional allow-list for admin users.

Deployment

Deployment is intentionally boring: Docker Compose starts the bot, and a cron job checks for updates and rebuilds the container when needed. This homemade DevOps setup definitely has limitations, but is more than enough for an application like this.
With this a zero downtime deployment is possible, it will only have downtimes when a release is broken because a rollback takes time.
